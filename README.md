## 设计原则
### 可扩展性基本要求
- 可重构 业务层代码可测试
- MQ DB 等中间件可以在不动业务代码的前提下更换（实现适配器）
  
### 可扩展性高级要求
- 服务间(nestjs 里面体现为模块)互调，屏蔽远程调用和本地调用细节
- 可复用 流程引擎 模块引擎 规则引擎 等通用能力抽象出来

### 单机高性能基本要求
- 网络模型
- 进程模型
- 缓存
- 队列

### 单机高性能高级要求
- 流处理

### 集群高性能基本要求
- 批处理
- map-reduce 支持
